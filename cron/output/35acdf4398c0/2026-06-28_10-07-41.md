### Authentication by Endpoint Group (Critical Distinction)

| Endpoint Group | Auth Method | Header | Notes |
|----------------|-------------|--------|-------|
| **Items** (`/api/items/...`) | Tenant API Key | `X-Api-Key: ten_...` | Superuser-generated in Portal UI → Settings → API Keys |
| **Stats/Quality** (`/api/items/stats*`) | **Tenant API Key** | `X-Api-Key: ten_...` | **Works** despite docs saying CookieAuth |
| **Business Partners** (`/api/business-partners/bp*`, `/api/business-partners/contacts`) | Tenant API Key | `X-Api-Key: ten_...` | **Works** — same as Items module |
| **Accounts/Clients** (`/api/accounts`) | **Bearer JWT** | `Authorization: Bearer *** | User login token — X-Api-Key returns **401** |
| **Categories** (`/api/categories*`) | **Bearer JWT** | `Authorization: Bearer *** | User login token — X-Api-Key returns 404/401 |
| **Integrations** (`/api/...` on integrations domain) | ApiKey | `Authorization: ApiKey <key>` | Different service — see EZY Integrations docs |

**Critical**: The Items API and Accounts API use **different authentication schemes** on the same tenant domain. Items work with `X-Api-Key: ten_...`; Accounts require a user Bearer token from login.

#### Price Lists — **NOT MANAGEABLE VIA API**
- Price lists exist in separate `pricing-tax` service
- **Must be created in Portal UI**: Settings → Price Lists → New
- API only references them by `priceListCode` string on items
- **Discover priceListId**: `GET /api/pricing-tax/price-lists` (requires `X-Api-Key: ten_...`) — returns all lists with UUID id per code (e.g., `COST_HACIENDA → 1be181ec-...`)
- **Price masking**: API key without price permissions → `prices: []` (empty array) in response
- **List price lists** (tenant API key works): `GET /api/pricing-tax/price-lists` — returns `id`, `code`, `name`, `currencyCode` for all lists
- **Add/update item prices via PATCH**: `PATCH /api/items/items/{id}` with `prices` array. Includes `version` for optimistic concurrency. On success, `version` increments (1 → 2) confirming the write even when prices array is masked in the response. See `references/pricing-tax-endpoint.md` for full payload shape and Python helpers.

### Critical Patterns

**ItemCode Format Convention (vivero tenant)**
- Pattern: `PL-` + UPPERCASE_NAME_WITH_DASHES
- Examples: `PL-AGAVE-EN-POTE`, `PL-AGLONEMAS`, `PL-ALBAHACA`

**Price Masking**
- Every price row has `masked: boolean`
- `masked=true` → `price=null`
- Tenant API key lacks price permissions → `prices: []` (empty array)

**Optimistic Concurrency**
- `version` (integer) on Item — must match on PATCH or 409 conflict

**Pagination**
- Standard: `page`, `perPage` (max 100). Response: `total`, `hasMore`
- **Always paginate** — default 20, easy to miss items

**PATCH requires `version` for optimistic concurrency** — always fetch the item fresh before patching or you'll get a 409 Conflict. This applies to ALL PATCH calls including tag updates.

**Tagging items via PATCH**: Items support `tags: string[]`. Add labels like `pet_friendly` by PATCH with the current `version` (optimistic concurrency). On Jun 25 2026, 69 vivero items were tagged `pet_friendly` via this pattern — after removing 2 false-positive Philodendrons identified via ASPCA cross-reference (original count was 71). See `references/pet-friendly-classification.md` for the pet-safe plant list.

## Common Pitfalls
1. **Using `stockTotal`** → field does NOT exist. Use `itemStockTotal`.
2. **`initialUOMs` in POST causes `chk_uom_code` SQL constraint** → Omit `initialUOMs` entirely at creation. Use `baseUom: "EA"` without `initialUOMs`. Same fix applies if adding UOMs via PATCH fails with this error.
3. **`/api/item-groups` and `/api/uoms` return 404** → UOM groups not accessible via REST. Valid UOM codes discovered from portal items (ANTORCHA used `EA`). For new items, omit `initialUOMs` and rely on `baseUom: "EA"`.
4. **Creating Google Sheet tabs** → `sheets create` makes new spreadsheets. Use `spreadsheets().batchUpdate` with `addSheet` request body to add a tab to an existing spreadsheet.
2. **Bearer token on Items endpoint → 401**: Items module (`/api/items/...`) on ezyts.com tenants requires `X-Api-Key: ten_...` — NEVER `Authorization: Bearer`. Using Bearer returns 401 even with a valid token. The auth table lists BearerAuth but the vivero tenant specifically uses X-Api-Key for all Items operations. If you get 401, check the header name first.
3. **Forgetting `expand=prices`** → prices array missing entirely
4. **Missing `version` on PATCH** → 409 conflict
5. **PATCH with `prices` replaces entire array** → a PATCH that includes `prices` overwrites all existing prices, it does not merge. Fetch current prices first and include them all in the PATCH payload to preserve them.
6. **Not paginating** → default 20, easy to miss data
6. **Using `itemGroupId` in PATCH** → field is `itemGroup` (string UUID)
7. **Price Lists not manageable via API** → must use Portal UI
8. **No image upload endpoint** → must use Portal UI or direct storage API
9. **`/internal/items/{id}/stock` serves HTML, not API** → use Portal UI for stock adjustments
10. **urllib blocked by Cloudflare** → add Chrome User-Agent + SSL context
11. **Price masking hides prices array on PATCH success** → `prices: []` in response even when price was written (no price-read permission). Verify via Portal UI or by incrementing `version` in response (1 → 2 confirms success). See `references/pricing-tax-endpoint.md`
12. **Google Sheets `sheets update` wrong dimensions** → 2D array required: `[[val], [val], ...]`
12. **Sheet names from import may have typos** → e.g., "FITTONIA" vs "FITONIA". Fix cell directly.
13. **`pageSize` vs `maxRecords` in Airtable** → use `pageSize=100` not `maxRecords` for pagination.
14. **Normalize collision on similar items** → e.g., "CAJA POTE 120" and "CAJA POTE 180" both normalize to "CAJA POTE". Always verify mismatch by itemCode, not just normalized name.
15. **AppleScript Mail draft when cua-driver unavailable** → see `references/applescript-mail-draft.md`
16. **Python heredoc via terminal `cat > file <<'EOF'` fails** — when the Python code inside a heredoc contains `from` imports, nested lists/dicts, or the TOKEN_FILE path (with `*** ` prefix), the shell heredoc corrupts the content or crashes the shell parser. Fix: write the Python script to disk with `write_file` (which handles multi-line strings cleanly), then execute it with the hermes venv Python. Pattern: `write_file(path="/tmp/script.py", content=...)` + `terminal("/Users/abrahamkortovich/.hermes/hermes-agent/venv/bin/python3 /tmp/script.py")`.
17. **Price masking on PATCH response** — `prices: []` in the PATCH response does NOT mean the price wasn't written. Check the `version` field: if it incremented from N to N+1, the write succeeded. The tenant API key simply lacks permission to read prices back. Verify via Portal UI if needed.
